# Day 70: Git & Version Control

## Project Overview
This day covered Git and version control fundamentals. Since this entire 100-day challenge is already managed in a Git repository with a structured branching strategy (one branch per week), the concepts were already in practice throughout the challenge.

## Key Concepts Covered
- **Version Control**: Tracking changes to code over time with commits
- **Branching**: Isolating work into separate branches (e.g., `Week_1` through `Week_10`)
- **Merging**: Integrating changes from one branch into another
- **Remote Repositories**: Pushing and pulling code to/from GitHub
- **Commit History**: Writing meaningful commit messages to document progress
- **.gitignore**: Excluding sensitive files (`.env`, `instance/`, `.venv`) from version control

## Already Applied Throughout This Challenge
- Weekly branches (`Week_1` to `Week_10`) for organized, isolated progress
- Consistent commit messages following the pattern: `Day_XX Added: <project description>`
- `.gitignore` configured to exclude virtual environments, databases, and API keys
- Environment variables used across projects (Days 29, 33, 35, 46, 47, 64, 66) to keep secrets out of version control
